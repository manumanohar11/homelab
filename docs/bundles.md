# Bundles

[Back to README](../README.md)

Bundles keep the beginner path small while leaving the full homelab available.

## Media

Adds Gluetun, the *Arr stack, downloaders, request apps, and alternate media services.

```bash
make init BUNDLES="media"
make up BUNDLES="media" PROFILES="arr jellyfin"
```

Uses `env/bundles/media.env.example`.

Required before first start:

- VPN provider credentials: `VPN_SERVICE_PROVIDER`, `OPENVPN_USER`, `OPENVPN_PASSWORD`
- Host paths for media and downloads under `${DOCKER_MEDIA_DIR}`

Good host prep:

- Create `${DOCKER_MEDIA_DIR}/downloads` before enabling `arr` or `downloaders`
- Test `make config BUNDLES="media" PROFILES="arr"` before first startup

Common profile combinations:

- `arr`: Radarr, Sonarr, Lidarr, Readarr, Bazarr, Whisparr, Recyclarr, qBittorrent, Prowlarr, Bitmagnet, Overseerr
- `jellyfin`: Jellyfin and Jellyseerr
- `kavita`, `navidrome`, `stash`, `tdarr`, `maintainerr`, `notifiarr`

## Apps

Adds productivity, document, knowledge, file-sharing, and automation services.

```bash
make init BUNDLES="apps"
make up BUNDLES="apps"
make up BUNDLES="apps" PROFILES="files automation kasm"
```

Uses `env/bundles/apps.env.example`.

Required before first start:

- `SEARXNG_SECRET` (generated by `make init BUNDLES="apps"` unless already set)
- App secrets and database passwords for Joplin, Paperless-ngx, Karakeep, Docmost, and ERPNext

Good host prep:

- Create `${PAPERLESS_CONSUME_DIR}` if you plan to use Paperless ingestion
- Plan space under `${DOCKER_BASE_DIR}/erpnext` before enabling ERPNext because the sites and MariaDB data live there
- Review which unprofiled apps you actually want exposed before bringing up the whole bundle

Unprofiled apps in this bundle include SearXNG, Syncthing, Joplin, Paperless-ngx, Stirling PDF, Karakeep, Docmost, and ERPNext.

## Ops

Adds monitoring, logging, diagnostics, and advanced backup endpoints.

```bash
make init BUNDLES="ops"
make up BUNDLES="ops" PROFILES="monitoring"
make up BUNDLES="ops" PROFILES="monitoring dashboard speedtest scrutiny"
```

Uses `env/bundles/ops.env.example`.

Required before first start:

- `GRAFANA_ADMIN_PASSWORD`

Good host prep:

- Plan disk space for Prometheus and Loki data under `${DOCKER_BASE_DIR}`
- Run `make sync-config` only after changing tracked monitoring/logging templates

Base ops services include Prometheus, Alertmanager, Grafana, Node Exporter, Uptime Kuma, Loki, and Promtail.

## Access

Adds Pangolin/Newt remote access.

```bash
make init BUNDLES="access"
make up BUNDLES="access"
```

Uses `env/bundles/access.env.example`.

Required before first start:

- `NEWT_ID` and `NEWT_SECRET` for the tunnel agent

Good host prep:

- Validate the Pangolin endpoint and confirm the local Docker host can reach it
- Run `make config BUNDLES="access"` before first startup

`newt` is the only tracked service in the access bundle.
